# ✅ Чек-лист проверки бота

Быстрая проверка работоспособности после создания бота.

## 🔴 Быстрый чек (5 минут)

### 1. Конструктор [2 мин]

- [ ] Откройте `index.html` в браузере
- [ ] Вкладка **"⚙️ Настройки"**
- [ ] GitHub токен введён
- [ ] Repository: `coopbot`
- [ ] Username: `kkav45`
- [ ] Нажмите **"Подключить"** → ✅ Успешно

### 2. Бот [1 мин]

- [ ] Вкладка **"📊 Мои боты"**
- [ ] Бот создан
- [ ] Сценарий `/start` существует
- [ ] Нажмите **"🚀 Опубликовать"**

### 3. GitHub [1 мин]

- [ ] Откройте: https://github.com/kkav45/coopbot/tree/main/bots
- [ ] Папка с ботом видна
- [ ] Файлы внутри:
  - [ ] `bot-config.json`
  - [ ] `bot.py`
  - [ ] `requirements.txt`
  - [ ] `.env.example`

### 4. Telegram [1 мин]

- [ ] Откройте @BotFather
- [ ] `/mybots` → ваш бот
- [ ] Токен скопирован
- [ ] Откройте вашего бота
- [ ] `/start` → бот отвечает ✅

## 🟡 Полная проверка (15 минут)

### 5. Локальный запуск [5 мин]

```bash
# Клонируйте
git clone https://github.com/kkav45/coopbot.git
cd coopbot/bots/ваш_бот

# Создайте .env
cp .env.example .env

# Откройте .env и вставьте токен
# BOT_TOKEN=123456789:ABCdef...

# Установите зависимости
pip install -r requirements.txt

# Запустите
python bot.py
```

**Ожидаемый результат:**
```
🤖 Бот 'Имя бота' запущен...
Username: @ваш_бот
```

## 🚨 Если что-то не работает

### Бот не публикуется в GitHub

```
❌ Ошибка: Bad credentials
✅ Решение: Пересоздайте токен в GitHub settings

❌ Ошибка: Repository not found
✅ Решение: Проверьте имя репозитория (case-sensitive)
```

### Бот не запускается локально

```
❌ ModuleNotFoundError: No module named 'aiogram'
✅ Решение: pip install -r requirements.txt

❌ BOT_TOKEN not found
✅ Решение: Проверьте .env файл (без пробелов, правильный формат)
```

### Бот не отвечает в Telegram

```
❌ Бот молчит
✅ Решение:
  1. Проверьте токен в .env
  2. Проверьте логи бота
  3. Убедитесь, что бот запущен (python bot.py)
  4. Проверьте интернет-соединение
```

### Сценарии не работают

```
❌ Бот отвечает не то
✅ Решение:
  1. Проверьте bot-config.json
  2. Убедитесь, что триггеры совпадают
  3. Перезапустите бота
```
